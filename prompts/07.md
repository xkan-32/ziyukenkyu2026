# ドキュメントの目的

このドキュメントは、実装者向けの詳細手順書ではありません。
ChatGPTに現状をレビューしてもらうための「実装状況サマリー」です。

以下の観点が一目で分かるようにしてください。

* 何が実装済みか
* 何を実機確認済みか
* 何が未実装か
* 何が未テストか
* 設計思想とズレていそうな箇所があるか
* 次にやるべきことは何か

# 書き方のルール

* 箇条書き中心で簡潔に書く
* ただし、判断に必要な事実は省略しない
* 推測と事実を分ける
* 「実装済み」「未実装」「未確認」を明確に分ける
* コードの全文は貼らない
* 代表的なJSONレスポンスは必要最小限にする
* 設計思想とのズレが疑われる箇所は、隠さず「要確認」として書く
* 実装が存在しないものを実装済みのように書かない
* 実機未確認のものを確認済みのように書かない

# 必ず含める内容

## 全体方針

以下の方針が現在も守られているか確認して書いてください。

* AI側の小松菜栽培ログを保存するシステムである
* 人間側の判断・作業・写真・収穫結果はシステムに保存しない
* Raspberry Pi は観測・通信・ローカルスケジュール担当
* Arduino は安全制御付き実行装置
* Cloud Run / Gemini は判断担当
* 定期観測とAI判断は分離する
* 定期的な土壌水分測定はデータ蓄積目的
* 乾燥しているだけでは自動水やりしない
* 水やり判断は将来、朝・夕方のAI判断に従う
* Arduinoは「水をあげるべきか」は判断しない
* Arduinoは `water <duration_ms>` を受けた場合に、安全実行可否だけを判定する

## 現時点のリスク・注意点

以下を必ず含めてください。

* 土壌水分センサー校正値は暫定であり、実土壌で再校正が必要
* 水道接続前に流量調整バルブと手動バルブが必要
* 流量測定は未実施なら未実施と書く
* Raspberry Pi実装前は定期観測は未実施
* AI判断前は自動水やり判断は未実装
* 12V系とArduino 5V系の混同に注意
* 緊急時は手動バルブと12V電源遮断が優先
