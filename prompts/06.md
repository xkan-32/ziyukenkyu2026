水道・給水にはまだ接続しない前提です。まずは電気的なON/OFF確認と、電磁バルブ単体の「カチッ」確認までを行います。

配線前提:
- Arduino VALVE_PIN / D2 -> MOSFETモジュール HIGH/PWM
- Arduino GND -> MOSFETモジュール GND
- 12V電源 + -> MOSFETモジュール VIN+
- 12V電源 - -> MOSFETモジュール VIN-
- 電磁バルブ + -> MOSFETモジュール OUT+
- 電磁バルブ - -> MOSFETモジュール OUT-
- LOW-SWITCH は今回は使用しない
- 水道・給水にはまだ接続しない

重要な安全方針:
- Arduino GPIOから12Vバルブを直接駆動しない
- 12Vの+をArduinoへ入れない
- 水を接続する前に必ず電気的動作確認をする
- 実バルブ接続中に異常があればすぐ12V電源を切る
- `close` コマンドで確実にOFFになることを確認する
- `water 100000` が MAX_SINGLE_WATER_MS に切り詰められることを確認する
- wet状態で `water` が拒否されることを確認する
- 作業中に安全ロジックを弱める変更はしない

今回の目的:
1. 現在のコードをコンパイルする
2. Arduinoに書き込む
3. dry-run mode で既存動作を再確認する

# 作業手順

## 1. 現状確認

まず以下を確認してください。

- config.h の現在値
- DRY_RUN_MODE
- VALVE_PIN
- VALVE_ACTIVE_HIGH
- MAX_SINGLE_WATER_MS
- MAX_DAILY_WATER_MS
- WET_REJECT_PERCENT
- SOIL_SENSOR_DRY_RAW
- SOIL_SENSOR_WET_RAW

この段階ではコード変更しないで、現在値を把握してください。

## 2. コンパイル

以下を実行してください。

```bash
cd edge/arduino/valve_controller
make compile
````

コンパイルエラーが出た場合:

* エラー内容を確認
* 最小限の修正だけ行う
* 安全ロジックを弱める変更はしない
* 修正した場合は理由を記録する

## 3. dry-run mode の確認

まず `DRY_RUN_MODE=true` の状態で確認してください。

必要なら config.h を一時的に `DRY_RUN_MODE=true` にしてください。

実行:

```bash
make upload
make monitor
```

シリアルモニタで以下を確認してください。

```text
status
read
water 500
close
water 100000
```

確認ポイント:

* 起動時に valve_open=false である
* `status` が JSON で返る
* `read` が JSON で返る
* `water 500` でLEDが短く点灯する
* `close` でLEDがOFFになる
* `water 100000` が MAX_SINGLE_WATER_MS に切り詰められる
* すべて1行JSONで返る

この時点で異常があれば、DRY_RUN_MODE=false には進まないでください。

## 4. MOSFETモジュール出力確認

dry-run が問題なければ、次に実バルブ制御モードへ進みます。

config.h を以下に変更してください。

```cpp
#define DRY_RUN_MODE false
```

また、通常は以下のままでよいです。

```cpp
#define VALVE_ACTIVE_HIGH true
```

変更後にコンパイル・書き込みしてください。

```bash
make compile
make upload
make monitor
```

この段階では、水道・給水には接続しないでください。

まずシリアルモニタで以下を実行してください。

```text
status
close
water 500
close
```

確認ポイント:

* `status` で dry_run=false になっている
* `close` で valve_open=false になる
* `water 500` 実行中だけ MOSFET 出力がONになる
* 電磁バルブを接続済みなら、実行中だけ「カチッ」と動作する
* 実行後は必ずOFFになる

もし `water 500` でバルブが常時ON、または `close` で止まらない場合:

1. すぐに12V電源を切る
2. 配線を確認する
3. VALVE_ACTIVE_HIGH を反転する必要があるか検討する
4. ただしコード変更前に原因を記録する

## 5. 電磁バルブ単体テスト

水はまだ接続しません。

以下を順番に実行してください。

```text
close
water 300
close
water 500
close
water 1000
close
```

確認ポイント:

* それぞれ指定時間だけ「カチッ」と動作する
* `close` 後にバルブがOFFになる
* 連続実行しても異常発熱や異音がない
* MOSFETモジュールが極端に熱くならない
* 12V電源に異常がない

## 6. 安全ロジック確認

以下を確認してください。

### 単回最大開放時間

```text
water 100000
```

期待:

* actual_duration_ms が MAX_SINGLE_WATER_MS に切り詰められる
* applied_limit=true になる
* 実行後に valve が閉じる

### wet拒否

土壌水分センサーを湿らせた状態で実行してください。

```text
read
water 500
```

期待:

* read で is_wet=true または moisture が WET_REJECT_PERCENT 以上になる
* water が rejected_by_safety / soil_too_wet で拒否される
* actual_duration_ms=0
* バルブは開かない

### 不正コマンド

```text
abc
water
water abc
```

期待:

* error JSON が返る
* バルブは閉じたまま

## 9. 問題が出た場合の対応

### バルブが開きっぱなしになる

1. すぐ12V電源を切る
2. Arduino側で `close` を送る前に、物理的に12Vを遮断する
3. 配線を再確認する
4. HIGH/PWM と LOW-SWITCH の取り違えがないか確認する
5. VALVE_ACTIVE_HIGH の設定を確認する
6. 原因と対応を TEST_LOG.md に記録する

### バルブが全く動かない

確認:

* 12V電源が入っているか
* VIN+ / VIN- が正しいか
* OUT+ / OUT- にバルブが接続されているか
* Arduino D2 が HIGH/PWM に入っているか
* Arduino GND とMOSFET制御GNDが共通か
* DRY_RUN_MODE=false になっているか
* wet拒否や日次上限拒否で実行されていないか
* `status` の dry_run / daily_watered_ms を確認する

### water が拒否される

確認:

* soil_too_wet ではないか
* daily_limit_exceeded ではないか
* invalid_duration_ms ではないか
* センサー校正値が暫定値のため wet 判定になりすぎていないか

必要に応じて、テスト目的で WET_REJECT_PERCENT を一時的に上げてもよいですが、変更理由を記録してください。
ただし安全ロジック自体を削除しないでください。
