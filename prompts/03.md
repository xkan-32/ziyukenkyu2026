# Arduinoの責務

Arduinoは単なる命令実行装置ではなく、水回りの安全制御装置として扱います。

実装する責務は以下です。

- 起動時に必ずバルブを閉じる
- 土壌水分センサーの raw 値を取得する
- dry/wet 校正値から soil_moisture_percent に変換する
- wet状態では水やり命令を拒否する
- 1回あたりの最大バルブ開放時間を強制する
- 1日あたりの累積バルブ開放時間上限を強制する
- 不正コマンドは拒否する
- 異常時、拒否時、不正コマンド時は必ずバルブを閉じる
- すべてのレスポンスをJSONで返す

# 実装するシリアルコマンド

シリアルモニタまたは将来のRaspberry Piから、1行1コマンドで受け付けます。

## 1. read

現在の土壌水分を読み取り、JSONで返す。

レスポンス例:

{
  "status": "ok",
  "command": "read",
  "soil_moisture_raw": 612,
  "soil_moisture_percent": 42.5,
  "is_wet": false
}

## 2. status

現在のデバイス状態をJSONで返す。

含める項目:

- uptime_ms
- valve_open
- dry_run
- daily_watered_ms
- max_single_water_ms
- max_daily_water_ms
- wet_reject_percent
- soil_moisture_raw
- soil_moisture_percent
- is_wet

レスポンス例:

{
  "status": "ok",
  "command": "status",
  "uptime_ms": 123456,
  "valve_open": false,
  "dry_run": true,
  "daily_watered_ms": 0,
  "max_single_water_ms": 5000,
  "max_daily_water_ms": 30000,
  "wet_reject_percent": 75.0,
  "soil_moisture_raw": 612,
  "soil_moisture_percent": 42.5,
  "is_wet": false
}

## 3. close

バルブを強制的に閉じる。
dry-run mode の場合も、テストLEDをOFFにする。

レスポンス例:

{
  "status": "ok",
  "command": "close",
  "valve_open": false,
  "message": "valve_closed"
}

## 4. water <duration_ms>

指定ミリ秒だけバルブを開く。
ただし、安全判定を必ず先に行う。

処理順序:

1. duration_ms の形式チェック
2. duration_ms が 0 以下なら拒否
3. 実行前に土壌水分を読む
4. wet 判定なら rejected_by_safety として拒否
5. 単回最大開放時間を超える場合は MAX_SINGLE_WATER_MS に切り詰める
6. 1日累積上限を超える場合は rejected_by_safety として拒否
7. dry-run mode の場合は、実バルブ制御ではなくTEST_LED_PINだけをON/OFFする
8. dry-run mode でない場合は、MOSFETモジュール制御ピンをON/OFFする
9. 実行後に必ず closeValve() を呼ぶ
10. 実行後に土壌水分を再読取する
11. JSONで結果を返す

成功レスポンス例:

{
  "status": "ok",
  "command": "water",
  "moisture_before_percent": 41.2,
  "moisture_after_percent": 54.8,
  "requested_duration_ms": 3200,
  "actual_duration_ms": 3200,
  "applied_limit": false,
  "dry_run": true,
  "message": "watered"
}

単回上限で切り詰めた場合:

{
  "status": "ok",
  "command": "water",
  "moisture_before_percent": 41.2,
  "moisture_after_percent": 43.0,
  "requested_duration_ms": 10000,
  "actual_duration_ms": 5000,
  "applied_limit": true,
  "dry_run": true,
  "message": "watered_with_limit"
}

wet拒否レスポンス例:

{
  "status": "rejected_by_safety",
  "command": "water",
  "reason": "soil_too_wet",
  "moisture_before_percent": 82.1,

# 土壌水分パーセント換算

raw値はセンサーや配線により、乾燥時に大きくなる場合と小さくなる場合があります。

今回の初期実装では、DRY_RAW と WET_RAW の大小関係に依存しすぎないようにしてください。

- DRY_RAW と WET_RAW から 0〜100% に map する
- 0未満は0に丸める
- 100超過は100に丸める
- DRY_RAW == WET_RAW の場合は安全にエラー扱いまたは0%扱いにする

# バルブ制御

実装する関数例:

- setupPins()
- closeValve()
- openValve()
- readSoilRaw()
- readSoilPercent()
- isWet()
- handleRead()
- handleStatus()
- handleClose()

- printJson...()

注意:

- setup() の最初の方で必ず closeValve() を呼ぶ
- loop() でSerialから1行読み取って処理する
- コマンド処理中も、不明な場合は closeValve() を呼ぶ
- water実行後は必ず closeValve() を呼ぶ
- dry-run mode の場合、VALVE_CONTROL_PINを操作せずTEST_LED_PINのみ操作してもよい
- dry-run mode でない場合、VALVE_CONTROL_PINでMOSFETモジュールを制御する
- VALVE_ACTIVE_HIGH が false の場合にも対応する

# MOSFETモジュール配線前提

README.mdに以下を明記してください。

- Arduino GPIO から MOSFETモジュールの IN / SIG / PWM 端子へ接続する
- Arduino GND と MOSFETモジュールの制御GNDを共通にする
- 12V電源はバルブ駆動用として別系統にする
- 電磁バルブは12V側に接続する
- Arduinoピンから12V電磁バルブを直接駆動してはいけない
- MOSFETモジュール上にフライバックダイオードが無い場合、電磁バルブ両端に外付けダイオードを追加する
- フライバックダイオードの向きは、カソードを12V+側、アノードをMOSFET/バルブマイナス側にする
- PWM制御は今回は使わず、ON/OFFのみとする
- 実バルブ接続前にdry-run modeでLED点灯確認を行う
- 水を接続する前に、テスターまたはLEDでON/OFF確認を行う

# 安全チェックとしてREADMEに必ず書くこと

- 手動バルブを上流に入れる
- 流量調整バルブを入れる
- 電磁バルブは常時閉タイプを使う
- 実験外時間は手動バルブを閉じる
- 長時間不在時は12V電源も落とす
- 起動時にバルブが閉じることを確認する
- wet状態で water が拒否されることを確認する
- 単回最大開放時間で必ず止まることを確認する
- 1日累積上限で拒否されることを確認する
- USBを抜いた場合、バルブが閉じる構成になっていることを確認する
- 水やりが止まらない場合は、まず手動バルブを閉め、次に12V電源を切る
