# プロジェクト概要

プロジェクト名は `komatsuna-ai-agent` です。

夏休み自由研究として、AIと人間が小松菜栽培で対決する実験を行います。

AI側は、カメラ画像、土壌水分センサー値、天気、過去ログをもとにLLMが判断し、水やりや作業指示を行います。

人間側は、息子が図書館、農家さん、園芸店、観察記録などをもとに判断します。

親はAI側の作業者・実装者であり、栽培判断には関与しません。

クラウド基盤は GCP を利用します。
インフラは Terraform による IaC で管理します。
AI判断には Vertex AI Gemini を利用します。
開発では Cursor / Codex を活用しますが、主にCodexで実装を進めます。

* 起動時・通信断時・異常時のバルブ閉制御
* Raspberry PiとのUSBシリアル通信

Arduinoは単なる実行装置ではなく、水回りの安全装置として扱います。

# ソレノイドバルブ仕様

使用するソレノイドバルブ仕様は以下です。

* バルブタイプ：2ウェイノーマルクローズ
* 通電時に開く
* サービス媒体：空気・ガス・液体・水
* 操作モデル：直動式
* フローモデル：単方向
* 流量：0.23Cv
* 応答時間：20ms未満
* 本体材質：アルマイト
* シール材質：NBR
* コイル電力：6.5W
* コイルクラス：Fクラス IP65
* デューティ：100%ED
* 動作圧力：最大115PSI
* 動作温度：-5〜80℃
* ポートサイズ：1/4インチNPTメスポート
* コイル電圧：12V DC
* 電気接続：12インチリード線

電流目安：

```text
6.5W ÷ 12V = 約0.54A
```

そのため、ソレノイド用電源は 12V 2A を推奨します。

制御は、初期実装では1chリレーモジュールを想定します。
最終的にはMOSFET制御も検討可能ですが、まずは確実性重視でリレーを前提に設計してください。

# 実験設計

栽培対象は小松菜です。

7月上旬から8月末までの約2か月を想定します。

2回戦方式とします。

* 1回戦：7月上旬〜8月上旬
* 改善期間：8月上旬
* 2回戦：7月下旬〜8月下旬

2回戦では、AIはファインチューニング、蒸留、専用学習を行いません。
1回戦の観察ログ、判断ログ、結果をプロンプトとしてLLMに渡し、改善案を出します。

人間側も1回戦の結果をもとに改善します。

評価項目は以下を想定します。

* 合計収穫重量
* 平均株重
* 草丈
* 葉の枚数
* 虫食いの少なさ
* 見た目
* 味
* 1回戦から2回戦への改善度

# 重要な追加要件：効果測定

このシステムは、AIが判断・実行して終わりではありません。

AIの判断や自動処理の後に、必ず効果測定を行います。

特に水やりについては、AIが「何ml水やりする」と判断し、Arduino側でそれをソレノイドバルブ開放秒数に変換します。

水やり後は、土壌水分量の変化を定期的に測定します。

測定ルールは以下です。

* 水やり直後〜1時間：5分間隔で土壌水分量を測定
* 水やり1時間経過後：1時間ごとに土壌水分量を測定
* 通常時：1時間ごとに測定

このログを蓄積することで、以下を分析します。

* 指定水量と土壌水分上昇量の関係
* 土への浸透速度
* 乾燥速度
* 天気・気温・湿度との関係
* 土の選定に必要な情報
* AIの水やり判断が適切だったか

この効果測定は重要な研究要素なので、設計書に明確に反映してください。

# 重要な追加要件：人間への指示と完了確認

AIが人間へ作業指示を出すケースがあります。

人間への指示は、将来的にLINEを想定します。

例：

* 間引きをしてください
* 防虫ネットを開けて葉の裏を確認してください
* 作業後の写真を送ってください
* 収穫してください
* 見た目や虫食いを確認してください

人間がLINEで完了連絡をしたら、AIエージェントは以下を行います。

1. 完了報告を受信
2. 作業後画像または報告テキストを取得
3. 指示内容が実行されたかをAIが判定
4. Firestoreへ実行確認ログを保存
5. 必要なら再撮影や追加作業を依頼

AIの確認結果は、以下のようなステータスで扱います。

* confirmed
* partially_confirmed
* not_confirmed
* unable_to_determine

この「人間への指示 → 完了報告 → AIによる実行確認」も詳細設計に含めてください。

# GCP構成

GCPでは以下を利用します。

* Cloud Storage：画像保存
* Firestore：センサー値、判断ログ、作業ログ、収穫結果
* Cloud Run：AI判断API、LINE Webhook、作業確認API
* Cloud Scheduler：定期実行
* Vertex AI Gemini：画像＋テキストによる判断
* Secret Manager：LINEトークンや外部APIキー
* Artifact Registry：Cloud Runコンテナ保存
* Cloud Logging：ログ確認
* Terraform：IaC管理

常時起動VMは利用しません。

水やり後5分間隔などの細かい測定は、Cloud SchedulerではなくRaspberry Pi側のローカルスケジューラで実行します。
理由は、ネットワーク断でも測定でき、デバイス近くで完結するためです。
