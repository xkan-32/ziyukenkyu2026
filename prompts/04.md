# 1. safety.h の日次上限判定を安全側に修正

現在のような unsigned long の引き算でアンダーフローし得る実装を避けてください。

修正方針:

inline bool wouldExceedDailyLimit(unsigned long currentDailyMs, unsigned long additionalMs) {
  if (currentDailyMs >= MAX_DAILY_WATER_MS) {
    return true;
  }
  return additionalMs > (MAX_DAILY_WATER_MS - currentDailyMs);
}

同等の安全側実装であれば可です。

# 2. 土壌水分センサー読み取りを平均化する

sensor.h の readSoilReading() で analogRead() 1回だけではなく、複数回読んで平均値を使うようにしてください。

config.h に以下を追加してください。

- SENSOR_READ_SAMPLES
- SENSOR_READ_INTERVAL_MS
- AFTER_WATER_READ_DELAY_MS

初期値例:
- SENSOR_READ_SAMPLES = 10
- SENSOR_READ_INTERVAL_MS = 10UL
- AFTER_WATER_READ_DELAY_MS = 1000UL

要件:
- SENSOR_READ_SAMPLES が 0 の場合でも暴走しないよう、安全に1回読み取り扱いにする
- 各読み取りの間に SENSOR_READ_INTERVAL_MS だけ delay してよい
- 平均値を int raw として扱う
- percent換算と wet判定は既存方針を維持する
- DRY_RAW / WET_RAW の大小関係が逆でも破綻しない現在の考え方は維持する
- 0〜100% に丸める処理は維持する

# 3. water後の moisture_after 読み取り前に少し待つ

valve_controller.ino の handleWaterCommand() で、closeValve() 後、moisture_after を読む前に AFTER_WATER_READ_DELAY_MS だけ待ってください。

要件:
- closeValve() は先に実行する
- バルブを閉じた状態で待つ
- 待機後に readSoilReading() する
- この待機はArduino単体レスポンス用であり、本格的な効果測定は将来Raspberry Pi側で5分間隔にする、という説明をREADMEに書く

# 4. water コマンドの引数 trim

`water    1000` のようにスペースが複数あっても受け付けられるようにしてください。

方針:
- processCommand() または handleWaterCommand() で water の引数に trim() をかける
- `water` だけ、または `water    ` は invalid_duration_ms として返す
- 不正時は必ず closeValve() されること

README.md に「配線図」または「配線例」セクションを追加してください。

## dry-run mode の配線

- Arduino UNO R4 WiFi 単体で動作確認できる
- TEST_LED_PIN は LED_BUILTIN
- DRY_RUN_MODE=true の間は VALVE_PIN で実バルブを開かない
- `water 1000` でLEDが1秒点灯すること

## 土壌水分センサー配線

3ピン土壌水分センサー想定で記載。

- センサー VCC -> Arduino 5V
- センサー GND -> Arduino GND
- センサー AO / AOUT / SIG -> Arduino A0
- センサー DO がある場合、今回は使わない
- config.h の SOIL_MOISTURE_PIN と一致させる

## MOSFETモジュール制御側配線

- Arduino VALVE_PIN, 現在D2, -> MOSFETモジュール IN / SIG / PWM
- Arduino GND -> MOSFETモジュール制御側 GND
- PWM制御は使わず、ON/OFFのみ
- Arduino GPIOから12Vバルブを直接駆動してはいけない

## 12Vバルブ駆動側配線

MOSFETモジュールの端子名は商品や基板表記で異なる可能性があるため、代表例として書いてください。

例:
- 12V電源 + -> 電磁バルブ +
- 電磁バルブ - -> MOSFETモジュール OUT- / LOAD- / V-
- 12V電源 - -> MOSFETモジュール VIN- / GND
- 必要に応じて 12V電源 + -> MOSFETモジュール VIN+ / V+

注意:
- 実際の基板表記を確認してから接続する
- 12V系とArduino 5V系を混同しない
- 制御GNDが必要なモジュールでは Arduino GND と制御GNDを共通にする
- ただし12Vの+をArduinoへ入れない

## フライバックダイオード

READMEに以下を明記してください。

- 電磁バルブはコイル負荷なのでOFF時に逆起電力が出る
- MOSFETモジュール上に保護ダイオードがあるか確認する
- 無い場合は電磁バルブ両端に外付けフライバックダイオードを追加する
- ダイオードのカソード、線がある側を12V+側へ接続
- ダイオードのアノード、線がない側をMOSFET側/バルブ-側へ接続

ASCII図も追加してください。

例:

12V+ ---- 電磁バルブ ---- MOSFET側
  |                         |
  +----|<|------------------+
       diode
       カソード(線あり)は12V+側

# 7. README.md に水回りの配線/接続順を追記

以下の順番を推奨として書いてください。

給水元 -> 手動バルブ -> 流量調整バルブ -> 電磁バルブ -> チューブ/点滴ノズル -> プランター

注意:
- 手動バルブは必須
- 流量調整バルブも入れる
- 実験外時間、外出時、就寝時は手動バルブを閉じる
- 長時間不在時は12V電源も落とす
- 水やりが止まらない場合は、まず手動バルブを閉め、次に12V電源を切る

# 8. README.md にテスト手順を追記

以下を具体的に書いてください。

## Arduino単体 dry-run テスト

1. DRY_RUN_MODE=true を確認
2. Arduinoに書き込み
3. シリアルモニタを115200bpsで開く
4. 起動時JSONで valve_open=false を確認
5. `status` を実行
6. `read` を実行
7. `water 1000` でLEDが1秒点灯することを確認
8. `water 100000` で MAX_SINGLE_WATER_MS に切り詰められることを確認
9. センサーを湿らせて `water 1000` が soil_too_wet で拒否されることを確認
10. `close` でLED/バルブ制御がOFFになることを確認
11. 不正コマンドで error JSON が返り、バルブが閉じることを確認

## MOSFETモジュール接続後テスト

1. まだ水を接続しない
2. DRY_RUN_MODE=false に変更
3. バルブの代わりにLEDやテスターでMOSFET出力のON/OFFを確認
4. 短い `water 500` で出力がON/OFFすることを確認
5. 電磁バルブだけを接続し、水なしでカチッと動作するか確認
6. その後、流量調整バルブを絞った状態で水テストする
