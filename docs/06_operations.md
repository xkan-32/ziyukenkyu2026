# 運用の考え方

運用では「植物が元気か」だけでなく、「ログが比較可能な形で残っているか」を同じくらい重視します。

# 日次運用

- 観測データが 1 時間ごとに記録されているか確認する
- 通常時は定期的に土壌水分が記録されているか確認する
- Raspberry Pi 初期運用では、まず観測だけが安定して回ることを確認する
- 朝・夕方の AI 判断が実行されたか確認する
- 判断時刻以外の乾燥値だけで水やりが実行されていないことを確認する
- 水やりイベントがあった場合、効果測定 12 件がそろっているか確認する
- 水やりが実行された場合のみ、5 分間隔の効果測定が始まることを確認する
- Raspberry Pi / Arduino の稼働状態を確認する
- 異常イベント
  - 安全拒否
  - センサー異常値
  - 通信失敗
  - 再送キュー滞留
  を確認する
- 実験外時間に手動バルブを閉めたか確認する

# 週次運用

- Firestore / GCS の保存量を確認する
- カメラレンズの汚れを清掃する
- センサー位置ずれや腐食を確認する
- リレー配線、ホース接続、防水状態を確認する
- 研究ログをバックアップする

# 実験開始前チェック

- AI 側と人間側で条件がそろっているか確認する
- 人間側の確認結果や比較メモは、システムではなく自由研究ノートやスプレッドシートで管理する
- `experiment_rounds` と `planter_profiles` を登録する
- `ALLOW_WATER_COMMAND_FROM_PI` が意図した値か確認する
- `DRY_RUN_MODE` が意図した値か確認する
- センサー校正値が最新か確認する
- wet 閾値が実土壌で校正済みか確認する
- 流量係数が測定済みか確認する
- 手動バルブが閉まることを確認する
- 手動バルブをすぐ閉じられる配置か確認する
- 初回観測スケジューラと再送キューを確認する

# Raspberry Pi 初期運用段階

- 第 1 段階では観測だけを先に回す
- この段階では `ALLOW_WATER_COMMAND_FROM_PI=false` を維持する
- 取得対象は画像、土壌水分、必要に応じた天気、ローカル JSONL、再送キューに限定する
- AI 判断ログは保存してよいが、`action=water` が返っても通水しない
- 水道接続した本番散水へ進むのは、安全試験と校正完了後だけとする

# 水やり後チェック

- `watering_events.status` が `success` か確認する
- 5 分間隔測定が開始されたか確認する
- `watering_events.effect_measurement_status` が `in_progress` へ遷移したか確認する
- 1 時間後に通常観測へ戻ったか確認する
- 原則 12 件の高頻度測定がそろったか確認する
- 土壌水分が不自然に上がり続けていないか確認する
- 判断時刻以外の定期観測だけでは `watering_events` が増えていないことを確認する

# 収穫時の作業

- AI 側の収穫日を記録する
- 重量、写真、見た目、味の評価を記録する
- 第 1 回戦後は改善期間に入り、AI 改善案と人間の反省を整理する

# センサー校正

- `tools/calibrate_soil_sensor.py` を使う
- 乾燥時と飽水時の値を記録する
- `calibration_profiles` に保存する
- 校正値を Arduino と Raspberry Pi の両方で参照できる形にする
- Arduino 初期実装では `edge/arduino/valve_controller/config.h` の `SOIL_SENSOR_DRY_RAW` と `SOIL_SENSOR_WET_RAW` を手動更新する
- Arduino 側は `SENSOR_READ_SAMPLES` 回の平均値で判定するため、校正時も数回読んだ平均を基準に確認する

# バルブ流量測定

- `tools/measure_valve_flow.py` を使う
- 開放時間ごとの実測 mL を取り、線形近似またはテーブル化する
- `calibration_profiles.flow_rate_ml_per_sec` に保存する
- 実バルブ接続前は Arduino を `DRY_RUN_MODE=true` にして、`read`、`status`、`close`、`water 1000` をシリアルで机上試験する
- 実機モードへ切り替える前に `ALLOW_WATER_COMMAND_FROM_PI` も意図どおり無効のままか確認する

# 障害時運用

| 症状 | 対応 |
|------|------|
| 水やりが止まらない | 手動バルブを閉め、12V 電源を切り、`safety_events` を記録する |
| Arduino 無応答 | 水やりは中止し、USB 再接続とサービス再起動を行う |
| Pi がネットワーク断 | ローカルキューへ保存し、回復後に再送する |
| Cloud Run 障害 | 観測継続、AI 判断は保留、手動監視へ切り替える |
| 再送キューが詰まる | `local_queue` の `failed` と `dead_letter` を確認し、原因を切り分ける |
