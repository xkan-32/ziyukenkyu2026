# プロジェクト概要

`komatsuna-ai-agent` は、夏休み自由研究に向けて AI 側の小松菜栽培エージェントを支えるシステムです。AI 側の判断、実行、効果をログとして残し、人間側の記録とはシステム外で比較します。

# このプロジェクトで大事にすること

- 比較できること
  - AI 側のログを外部の自由研究ノートや表と比較できること
- 後で説明できること
  - 判断理由、実行結果、効果を時系列で残す
- 安全であること
  - 水道直結バルブを扱うので、失敗しても危険側に倒れない
- 子どもの自由研究として分かりやすいこと
  - ただし実装者向けの技術詳細は省略しない

# 実験の骨子

- 対象作物は小松菜
- 期間は 2026 年 7 月上旬から 8 月末を想定
- 2 回戦方式で評価する
  - 第 1 回戦: 初回栽培
  - 改善期間: ログを見て改善
  - 第 2 回戦: 改善後に再戦
- AI はファインチューニングせず、1 回戦ログをプロンプトに再投入して改善する

# 全体構成

```text
[カメラ]      [土壌水分センサー]
    \            /
     \          /
      [Raspberry Pi] -- USB Serial -- [Arduino UNO R4 WiFi] -- Relay -- [12V NC Solenoid Valve]
            |
            +-- HTTPS --> [Cloud Run agent-api]
            +-- GCS ----> [Cloud Storage]
            +-- Firestore
            +-- Vertex AI Gemini
```

# コンポーネント責務

| コンポーネント | 主な責務 |
|------|------|
| Raspberry Pi | 撮影、観測統合、ローカルスケジュール、GCP 同期、Arduino 通信 |
| Arduino | センサー読み取り、バルブ制御、安全判定、危険命令の拒否 |
| Cloud Run `agent-api` | AI 判断、水やり効果分析、改善案、日次サマリー、エクスポート |
| Firestore | 観測、判断、実行、効果測定、収穫の構造化ログ保存 |
| Cloud Storage | AI側の元画像、推論用縮小画像、収穫画像、派生画像の保存 |
| Vertex AI Gemini | 画像＋テキストからの判断、水やり効果分析、改善案生成 |

# なぜこの分け方にするか

- Raspberry Pi は現場に近く、ネットワーク断でも観測と再送キューを維持できる
- Arduino はクラウドや Pi が誤っても危険な通水を拒否できる
- GCP は重い判断、永続化、エクスポートに向いている
- 水やり後 5 分ごとの測定は Cloud Scheduler よりローカル実行の方が確実

# 最重要フロー

- 観測フロー
  - 撮影、土壌水分取得、天気取得、保存
- AI 判断フロー
  - 観測を入力に、JSON で行動提案を返す
- 効果測定フロー
  - 水やり後 1 時間は 5 分間隔、その後は 1 時間間隔で測定する
- 研究比較フロー
  - AI 側ログを CSV やレポートとして出力し、人間側の記録とシステム外で比較する
